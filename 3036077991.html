MyBatis 是一款优秀的持久层框架，它支持自定义 SQL、存储过程以及高级映射。MyBatis 免除了几乎所有的 JDBC 代码以及设置参数和获取结果集的工作。MyBatis 可以通过简单的 XML 或注解来配置和映射原始类型、接口和 Java POJO（Plain Old Java Objects，普通老式 Java 对象）为数据库中的记录。

源码分析

获取SqlSessionFactory

1、new SqlSessionFactoryBuilder().build(inputStream); 构建SqlSessionFactory。

2、new XMLConfigBuilder(inputStream, environment, properties);创建构建Mybatis配置文件类，其内部使用了Configuration类来保存配置文件内容。

3、parser.parse();通过此方法开始解析mybatis.xml每个节点信息。

4、environmentsElement(root.evalNode(“environments”));此节点是解析数据源，一般来说的话mybatis.xml重点包含数据源和mapper。

5、mapperElement(root.evalNode(“mappers”));解析mappers节点。

6、mapperParser.parse();通过创建XMLMapperBuilder开始解析。

7、configurationElement(parser.evalNode(“/mapper”));解析每一个mapper.xml配置文件。

8、buildStatementFromContext(context.evalNodes(“select|insert|update|delete”));里面的话又是解析每一个子节点，由于只配置了CRUD所以进入到了SQL解析。

9、buildStatementFromContext(list, null);list入参是当前mapper.xml的所有sql语句标签。

10、statementParser.parseStatementNode();每条SQL语句会被XMLStatementBuilder进行解析。

11、builderAssistant.addMappedStatement();每条SQL语句的属性都会被封装成MappedStatement，然后通过namespace+方法名组成key，被保存到configuration的mappedStatements中。

12、bindMapperForNamespace();在解析完当前mapper.xml的节点之后，会对当前mapper进行保存。

13、configuration.addMapper(boundType);通过namespace反射出class,会被MapperRegistry里面的map所保存。

14、knownMappers.put(type, new MapperProxyFactory<>(type));MapperProxyFactory主要后期获取Mapper帮我们创建代理类，这里使用了JDK动态代理基于接口。

15、new DefaultSqlSessionFactory(config);绑定完Mapper之后最终返回SqlSessionFactory，configuration作用于SqlSessionFactory。

根据上面提供的配置文件最终会走的解析方法只有两个，environmentsElement(root.evalNode(“environments”));mapperElement(root.evalNode(“mappers”));

environment就是入参时说的数据源节点，这里如果为空的情况下会走environments节点属性的默认值，所以默认值要与其中另一一个子节点数据源的ID匹配，否则无法加载数据源到configuration中，其次就是两个数据源相同时就会进行顺序覆盖。

databaseId在解析mybatis配置的时候设置，通过这个方法databaseIdProviderElement(root.evalNode(“databaseIdProvider”));
DatabaseId是数据库厂商标识符，例如：mysql、oracle、sqlserver等。使用教程: databaseIdProvider

获取SqlSession

1、sqlSessionFactory.openSession();调用无参方法获取SqlSession。

2、openSessionFromDataSource(configuration.getDefaultExecutorType(), null, false);获取默认执行器SIMPLE不会自动提交事物。

3、configuration.getEnvironment();获取封装好的数据源。

4、getTransactionFactoryFromEnvironment(environment);获取事务工厂。

5、transactionFactory.newTransaction(environment.getDataSource(), level, autoCommit);获取事务。

6、configuration.newExecutor(tx, execType);获取执行器这是最重要。

7、new CachingExecutor(executor);通过执行器类型创建执行器，由于默认是SIMPLE所以会创建SimpleExecutor，但是因为二级缓存默认开启所以会创建CachingExecutor来装饰一级缓存。

使用无参的openSession方法获取SqlSession，configuration.getDefaultExecutorType()也就是默认执行器走的是SIMPLE。

  1. ExecutorType.SIMPLE：该类型的执行器没有特别的行为。它为每个语句的执行创建一个新的预处理语句。
  2. ExecutorType.REUSE：该类型的执行器会复用预处理语句。
  3. ExecutorType.BATCH：该类型的执行器会批量执行所有更新语句，如果 SELECT 在多个更新中间执行，将在必要时将多条更新语句分隔开来，以方便理解。

获取Mapper

1、sqlSession.getMapper(UserMapper.class);传入需要获取的Mapper的Class。

2、configuration.getMapper(type, this);configuration封装了一层获取Mapper的方法。

3、mapperRegistry.getMapper(type, sqlSession);我们的Mapper是保存在MapperRegistry这个类的Map下面。

4、(MapperProxyFactory) knownMappers.get(type);通过Class获取MapperProxyFactory。

5、mapperProxyFactory.newInstance(sqlSession);获取生成的动态代理对象。

6、new MapperProxy<>(sqlSession, mapperInterface, methodCache);封装我们Mapper对应的拦截器，后面动态生成的代理类需要继承此类。

7、Proxy.newProxyInstance(mapperInterface.getClassLoader(), new Class[] { mapperInterface }, mapperProxy);最后通过反射创建代理对象，最终返回给Mapper接口进行接收。

一级缓存和二级缓存的区别

  1. 一级缓存使用HashMap进行存储。
  2. 一级缓存作用于同一个SqlSession。
  3. 二级缓存作用于在同一个namespace。
  4. 二级缓存可以使用第三方缓存框架。

一级缓存存在什么问题

一级缓存存储数据在本地内存中，在集群情况下容易引发脏数据，举个栗子有两台集群机器（A、B）。
客户端在A中查询了一条记录，然后客户端在B中更新了A查询的记录，由于Mybatis清空缓存的条件是DML（增删改）操作，就会清空hashmap中缓存的所有数据。
所以A并不知情就出现了脏数据。

如何清空一级缓存

  1. SQL传入随机数 #{random} = #{random}。
  2. 执行DML操作。
  3. 调用SqlSession中的clearCache()方法。
