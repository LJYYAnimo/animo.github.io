SpringMvc回顾
发表于 2020-09-17| 更新于 2020-09-17
阅读量:

首先说到Web框架SpringMvc，那肯定是Spring的亲儿子，相对于底层实现那也是离不开最开始学习的Servlet。

Servlet

说到Servlet也并不陌生，定义一个Servlet相当于一个URL请求，然后需要在web.xml指定servlet和对应的Mapping映射路径。

Servlet类

继承HttpServlet然后实现对应的doGet、doPost方法，但是最终会走受保护的service，这里我们重写的是public service方法，里面会直接调用protected service方法，被重写的service方法会覆盖其他执行动作。

基于注解形式相对于web.xml配置更于简单不那么臃肿。

ServletContainerInitializer

这个类比较核心在SpringMVC中也是最为重要的主要用于加载第三方类到Servlet容器中。

@HandlesTypes注解参数是个数组Class，如果当前传入的类有子类则会把子类扫描到重载方法的set参数中，不包含父类。其次ServletContext里面可以添加Servlet、Filter、Listener。

实现了ServletContainerInitializer之后需要在当前目录下配置才能被Tomcat所加载，文件没有后缀里面的值就是实现类的全限定类名。

SpringMvc

Spring MVC属于SpringFrameWork的后续产品，已经融合在Spring Web Flow里面。Spring 框架提供了构建 Web 应用程序的全功能 MVC 模块。使用 Spring 可插入的 MVC 架构，从而在使用Spring进行WEB开发时，可以选择使用Spring的Spring MVC框架或集成其他MVC开发框架，如Struts1(现在一般不用)，Struts 2(一般老项目使用)等等。

此包已经包含了Spring、SpringMvc相关开发的Jar。其次不再使用web.xml进行启动Web项目而是采用Java配置类形式。

在Spring-Web包也有上面Servlet提到的文件里面也配置了这么一个初始化容器的类。

这里代码就和上面Servlet说的代码解析一模一样，我们的类只要继承了WebApplicationInitializer就会被扫描然后执行重写的onStartup方法，来启动Web项目。

这里的配置与web.xml的一模一样只是写成了Java代码，其次可以在网上对比其他基于web.xml的配置看一下就明白了。

如果后期自己集成SSM基于注解形式的话，这个类就可以当做Spring.xml，在之前基于XML的SSM中有很多XMl里面都是在注入Bean，其次可以改写成上面这种形式。

@EnableWebMvc开启注解形式，如果我们后期要写拦截器的话是需要注册到Web容器中的，WebMvcConfigurationSupport里面有个addInterceptors方法，但是我们的类是不能去继承WebMvcConfigurationSupport，因为@EnableWebMvc中已经注入了该类如果我们在继承的话会导致我们在里面重写的方法无效，也就导致了拦截器没注入。

以上对Servlet、SpringMvc的一些回顾，之前学Servlet挺懵逼的，现在回过头来再看看其实也就那么回事，在者就是Spring注入Bean的思想，如果从xml转化成Java配置类这个思想个人觉得十分重要。
